# Honda Service CMS

Сайт работает на обычном PHP-хостинге: PHP + MySQL, без Next.js, React и Node.js.

## Перед любыми действиями: резервная копия

1. Скачайте все файлы старого сайта через FTP или файловый менеджер Majordomo.
2. В панели Majordomo откройте phpMyAdmin.
3. Выберите базу старого WordPress-сайта.
4. Сделайте экспорт всей базы в SQL-файл.
5. Проверьте, что архив файлов и SQL-дамп открываются локально.

Старые таблицы WordPress (`wp_posts`, `wp_options`, `wp_users` и другие `wp_*`) не изменяются. Новый сайт использует только таблицы с префиксом `hs_new_`.

## Развёртывание

1. Загрузите файлы сайта в корень домена.
2. Убедитесь, что папка `/uploads` доступна для записи PHP.
3. В `config.php` укажите доступы к MySQL:
   - `HS_DB_HOST`
   - `HS_DB_NAME`
   - `HS_DB_USER`
   - `HS_DB_PASS`
4. В phpMyAdmin импортируйте `schema.sql`.
5. Создайте первого администратора.
6. Откройте `/admin`.

## Создание первого администратора

Создайте хэш пароля временным PHP-файлом или в локальной консоли:

```php
<?php
echo password_hash('CHANGE_THIS_PASSWORD', PASSWORD_DEFAULT);
```

Затем выполните SQL, подставив полученный хэш:

```sql
INSERT INTO hs_new_admin_users (username, password_hash)
VALUES ('admin', '$2y$10$PASTE_GENERATED_HASH_HERE');
```

После входа удалите временный PHP-файл, если создавали его на хостинге.

## Что хранится в MySQL

- SEO title/description.
- Контакты и ссылки.
- Тексты главной страницы.
- Услуги.
- Цены.
- Новости.
- Пути к изображениям и их статус.

## Что хранится в файлах

- CSS, JS, логотипы и базовые ассеты.
- Загруженные изображения в `/uploads`.

Физическое удаление изображений в админке не выполняется: изображения можно скрывать или заменять.
